“智农虫盾”项目研究报告 - 2026

“智农虫盾”：基于物联网与深度学习的猕猴桃病虫害监测预警系统研究

项目阶段：实践研发期（2024-2026）
主要针对作物：猕猴桃 (Actinidia chinensis)
摘要： 本文介绍了一种名为“智农虫盾”的智能化农业监测系统。针对猕猴桃种植过程中病虫害频发、人工监测成本高且滞后的问题，本研究提出了一种结合物联网（IoT）环境感知与卷积神经网络（CNN）图像识别的综合解决方案。系统通过收集农作物生长环境的实时参数与叶片图像，利用深度学习模型进行病害识别，并结合环境规律给出预警建议。目前，系统已完成基础框架搭建，实现了基于Flask的后端API服务及初步的CNN模型训练。
关键词：猕猴桃；病虫害监测；深度学习；物联网；Flask；智能农业

1. 引言

病虫害是影响农作物产量和品质的核心因素。特别是在猕猴桃产业中，溃疡病（Psa）、褐斑病等具有极强的传染性。传统的防治方法依赖人工巡检，存在误判率高、响应不及时等缺陷。本项目旨在创建高效、准确、实时的监测系统，为个体种植户和农业企业提供低成本、轻量化的技术支持。

2. 系统架构设计

2.1 技术路线

系统采用软硬件结合的架构。硬件侧负责环境参数（温湿度、光照）采集；软件侧基于Python开发，采用以下技术栈：

  • 后端框架：Flask (Python)
  • 深度学习库：TensorFlow / Keras
  • 图像处理：PIL, OpenCV
  • 数据处理：Numpy, Matplotlib

2.2 项目工程结构

目前项目已建立规范的模块化结构，如下表所示：

目录/文件 功能描述
data/ 存储原始图像数据集，按病害种类归类。
models/ 存放训练好的 .keras 模型文件及训练脚本。
utils/ 存放核心推理算法逻辑（如 predictor_smart.py）。
app.py 系统主程序，提供 Web 服务和 API 接口。
data_collector.py 自动化数据采集与目录初始化工具。

3. 关键模块实现

3.1 数据采集与预处理

通过 data_collector.py 构建标准数据集结构，支持包括健康、溃疡病、褐斑病、花腐病在内的四类核心识别目标。代码实现了目录自动生成逻辑，确保了数据训练的一致性。

3.2 模型训练逻辑

在 train_simple.py 中，系统采用了改进的 CNN 架构，引入了数据增强层（RandomFlip, RandomRotation）和批归一化（BatchNormalization），以提高模型在复杂田间环境下的泛化能力。模型输出采用 Softmax 激活函数进行多分类识别。

3.3 API 服务与业务逻辑

app.py 作为核心调度中心，实现了以下功能：

  • 支持多格式图片上传（PNG, JPG, GIF）。
  • 集成了 AI 预测器接口，并具备异常处理机制（AI不可用时自动回退至演示模式）。
  • 内置病害知识库（DISEASE_DB），为农户提供针对性的专家建议。

4. 创新点与特色

  1. 轻量化设计：系统旨在降低部署门槛，支持普通智能手机与基础传感器接入。
  2. 多维度预警：不仅依靠单一的图像识别，未来将整合病害变化规律与环境因子的相关性。
  3. 快速落地：专注于猕猴桃细分领域，具有较强的专业针对性。

5. 进度安排与预期成果

项目当前处于实践研发阶段。已初步构建好预测系统框架，并具备了病害基础知识储备。

  • 2024.11 - 2025.06： 完善模型精度，开发硬件采集终端。
  • 2025.07 - 2026.06： 系统集成测试，实地农园部署及研究报告结题。
